Each pull request must focus on a single, well-defined change; bundling multiple unrelated fixes or features in one PR is explicitly prohibited. CONTRIBUTING.md marks this as IMPORTANT and states it makes code review faster, increases acceptance likelihood, and maintains a clean git history. For features requiring many changes, contributors must reach out on Slack before opening a PR.
